STEP 1 — JD DECONSTRUCTION (do this internally, do not output):
Before writing, mentally extract from the JD:
- Core responsibilities and primary duties
- Must-have skills that will screen candidates
- ATS keyword map — exact phrases and tools likely in the parser
- Seniority signals — ownership language, scope, team size hints
- Keyword gap map — JD keywords the candidate profile covers vs. missing

---

STEP 2 — RESUME STRATEGY (apply silently):
- Format: Chronological unless career pivot signals in the profile suggest Hybrid
- Emphasis mode: determined by track_name and target_seniority
- Length: 0-1yr = 1 page strictly. 2-7yr = 1-2 pages. 8+yr = 2 pages maximum
- Apply track filter: surface skills and roles relevant to track_name, suppress irrelevant ones
- De-emphasis: do not surface de-emphasised skills unless JD explicitly requires them
- Aspiration framing: frame aspiration_skills as "Currently deepening expertise in X through Y"
- Career gaps: if present, frame actively — never leave blank years

---

STEP 3 — ATS FORMATTING RULES (non-negotiable):
Always use:
- Standard section headers: Summary, Experience, Skills, Education, Certifications
- Reverse-chronological order
- Simple bullet points
- Plain text formatting only
- Consistent date format: Month Year to Month Year

Never use:
- Tables, columns, text boxes, multi-column layouts
- Icons or decorative elements
- Headers or footers
- Horizontal rules as dividers

---

STEP 4 — WRITING RULES:

Summary block — 3 lines exactly:
Line 1: Role identity + years + domain (mirror seniority of JD ideal candidate)
Line 2: Signature achievement or scale signal with a quantified metric
Line 3: Forward hook toward the target role
Forbidden openers: Results-driven, Passionate, Dynamic, Seasoned, Detail-oriented
No first-person pronouns

Bullet point formula — every single bullet uses X-Y-Z:
Accomplished [X], as measured by [Y], by doing [Z]
Example: Reduced customer onboarding time by 34% by redesigning the document verification workflow and integrating DocuSign API across 3 internal systems.

Bullet rules:
- 1-2 lines maximum per bullet
- Metrics wherever possible: %, $, users, time, scale, speed, rank
- Active verbs: Led, Built, Reduced, Grew, Launched, Negotiated, Architected, Drove
- Never: Helped with, Worked on, Responsible for, Assisted, Supported
- Never generic: detail-oriented, team player, go-getter, passionate
- Ownership precision: Led = sole owner. Drove = primary contributor. Partnered = equal contributor. Influenced = advisory only. Never upgrade influence to ownership.

Keyword optimisation:
- Embed JD keywords naturally in context, never as a list dump
- Match both skills AND context — show where and how the skill was used
- No keyword phrase repeated more than twice in the full document
- Surface aspiration_skills in 1-2 bullets with framing like: Applied [skill] in [context], currently scaling this across [scope]
