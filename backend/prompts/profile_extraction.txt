FIELD RULES:

extracted_skills:
- Categorise into sub-arrays: technical, languages, frameworks, tools, methodologies, leadership
- Normalise to widely recognised form: Python not python, React not ReactJS
- Extract only skills that appear in the document and can be inferred from what is written
- Each sub-array may be empty if no skills in that category are found
- No minimum count — 3 real skills is better than 8 padded ones
- Also extract product management skills including: product strategy, roadmap planning, stakeholder management, user research, go-to-market, OKRs, agile, scrum, product analytics, A/B testing, requirements gathering, cross-functional leadership, executive communication, product launch, competitive analysis, customer discovery, PRD writing, sprint planning.

education_data:
- Extract all degrees and diplomas
- graduation_year: 4-digit integer if stated, otherwise null
- source_confidence: explicit if stated directly, inferred if calculated, unknown if absent

work_history:
- Extract every role including part-time, freelance, contract
- IMPORTANT: the documents above may include multiple versions of the same person's resume (e.g. tailored for different job applications), describing the SAME roles with different wording or emphasis. Treat roles that clearly refer to the same position (same company, same or near-same title, same or overlapping dates) as ONE role — do not duplicate them in the output.
- However, do NOT let this deduplication cause you to drop a genuinely DISTINCT role just because it only appears in some of the source documents (e.g. an earlier-career role that was trimmed from some tailored resume versions but appears in others, or only in a LinkedIn export). If a role appears in even one source document, include it.
- title: extract the CORE job title only — remove suffixes, team names, and project names
  - "Lead, Product Management - Productivity Insights" → "Lead Product Manager"
  - "Senior Software Engineer - Payments Platform" → "Senior Software Engineer"
  - "Data Analyst, JAPA Business Insights" → "Data Analyst"
  - "Associate Product Manager (APM)" → "Associate Product Manager"
  - Keep seniority level and function — remove everything else
- start_date and end_date: YYYY-MM format. If only year given use YYYY-01
- end_date: null for current roles, set is_current to true
- employment_type: one of full-time, part-time, contract, freelance, internship, board, unknown

years_of_experience:
- Calculate from earliest start_date to today
- Overlap rule: for parallel roles count overlapping period only once
- Exclude internships under 3 months
- Return as rounded integer

seniority_level:
- One of: intern, junior, mid, senior, lead, manager, director, vp, c-suite
- If ambiguous pick the lower one

extracted_summary:
- Third person, 2-3 sentences maximum
- Sentence 1: Role identity + years of experience + primary domain
- Sentence 2: Most significant verifiable achievement with metric if one exists in the document — never fabricate a metric
- Sentence 3 optional: Forward signal if stated in document
- No buzzwords or filler phrases

career_tracks:
- Distinct career tracks from work history as lowercase strings
- Maximum 3 tracks

certifications:
- Professional credentials listed separately from education
- year: integer or null
- Empty array if none found

phone:
- Extract the phone number if present anywhere in the document
- Include country code if present (e.g. +91 9866934483)
- Return null if not found — never fabricate a phone number

raw_profile_text:
- Format strictly as:
  ROLES: comma-separated list of Title at Company (Year-Year)
  SKILLS: comma-separated flat list of all skills across categories
  EDUCATION: comma-separated list of Degree, Field, Institution, Year
  ACHIEVEMENTS: up to 5 bullet points of strongest verifiable achievements with metrics where present
  GAPS/NOTES: any extraction uncertainties or data quality flags
- 200 words maximum

HALLUCINATION PREVENTION:
1. If a field cannot be populated from the document use its null/empty value
2. If a metric does not exist do not create one for extracted_summary
3. If a skill is implied but not stated do not include it
4. If graduation year requires more than one inferential step return null
